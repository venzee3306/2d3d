1.1 Authentication
Function / Property	Signature	Description
player	Player | null	Current authenticated player object, or null
isAuthenticated	boolean	Whether a player is logged in
login	(username: string, password: string) => boolean	Authenticates against mockPlayer. Sets player, balance, isAuthenticated. Returns true on success, false on failure. Valid credentials: aungaung / player123
logout	() => void	Clears player, isAuthenticated, balance, cart, bets, transactions. Resets all state to defaults
1.2 Language / i18n
Function / Property	Signature	Description
language	Language ('en' | 'mm')	Current active language
setLanguage	(lang: Language) => void	Switches language. Affects all t() calls
t	(key: string) => string	Translation lookup. Supports dot-notation keys (e.g. t('betNumber')). Returns the key itself if not found
1.3 Balance
Function / Property	Signature	Description
balance	number	Current player balance in MMK
updateBalance	(amount: number) => void	Adds amount to balance (negative to subtract). Also updates player.balance
1.4 Cart
Function / Property	Signature	Description
cart	CartItem[]	Current items in the betting cart
addToCart	(items: CartItem | CartItem[]) => void	Appends one or multiple items to cart
removeFromCart	(id: string) => void	Removes a single item by its id
updateCartItem	(id: string, amount: number) => void	Updates the amount of a specific cart item
clearCart	() => void	Empties the entire cart
cartTotal	number	Computed sum of all item.amount in cart
1.5 Bets
Function / Property	Signature	Description
bets	Bet[]	All placed bets (newest first)
activeBets	Bet[]	Computed: bets with status === 'Pending'
pastBets	Bet[]	Computed: bets with status !== 'Pending'
placeBets	(sessionId: string) => boolean	Converts entire cart into Bet[], deducts cartTotal from balance, creates a 'bet' transaction, clears cart. Returns false if cart is empty or balance insufficient
placeBetsDirectly	(items: CartItem[], sessionId: string) => boolean	Same as placeBets but takes items directly (does NOT use or clear cart). Used by PlayerBettingInterface. Returns false if items empty or total > balance
getBetsBySession	(sessionId: string) => Bet[]	Filters all bets matching a specific sessionId
placeBets internal flow:

Validate cart not empty, cartTotal <= balance
Map each CartItem → Bet with auto-generated id, current round (Morning if before 12, else Evening), status: 'Pending'
Prepend new bets to bets state
Call updateBalance(-cartTotal)
Call addTransaction(...) with type 'bet'
Call clearCart()
placeBetsDirectly internal flow: Same as above but uses provided items array instead of cart. Does NOT clear cart.

1.6 Transactions
Function / Property	Signature	Description
transactions	Transaction[]	All transaction records (newest first)
addTransaction	(transaction: Omit<Transaction, 'id'> & { id?: string }) => void	Creates a new transaction. Auto-generates id if not provided. Prepends to transactions
updateTransaction	(id: string, updates: Partial<Transaction>) => void	Merges updates into the transaction matching id. Used to change status: 'pending' → 'approved'
1.7 Results
Function / Property	Signature	Description
dailyResults	DailyResult[]	30 days of mock daily results (2D morning+evening + 3D every 5 days)

1.8 Payout Rates
Function / Property	Signature	Description
payoutRates	PayoutRate[]	Array of payout rate configs
getPayoutRate	(gameType: '2D' | '3D') => PayoutRate | undefined	Returns the payout rate for the given game type. 2D = 85x, 3D = 500x
1.9 Blocked Numbers

2.1 Player
interface Player {
  id: string;              // e.g. 'player-001'
  name: string;            // English name
  nameMM: string;          // Myanmar Unicode name
  username: string;        // Login username
  password: string;        // Login password (mock)
  phoneNumber: string;     // e.g. '09123456789'
  balance: number;         // Current balance in MMK
  agentId: string;         // Linked agent ID
  status: 'active' | 'suspended';
  createdAt: string;       // ISO timestamp
}
2.2 Bet
interface Bet {
  id: string;                          // Auto-generated: 'bet-{timestamp}-{random}'
  playerId: string;
  playerName: string;
  gameType: '2D' | '3D';
  betNumber: string;                   // The number bet on (e.g. '47', '123')
  amount: number;                      // Bet amount in MMK
  round: 'Morning' | 'Evening';       // Determined by time of placement
  placedAt: string;                    // ISO timestamp
  status: 'Pending' | 'Won' | 'Lost';
  winAmount?: number;                  // Only if Won
  sessionId?: string;                  // Groups bets from same submission
  pattern?: {                          // If placed via pattern
    type: PatternType;
    input: string;
    label: string;
    labelMM: string;
  };
}
2.3 CartItem
interface CartItem {
  id: string;                    // Unique per item
  gameMode: '2D' | '3D';
  number: string;                // The number to bet on
  amount: number;                // Bet amount
  pattern?: {                    // Optional pattern metadata
    type: PatternType;
    input: string;
    label: string;
    labelMM: string;
  };
}

interface Transaction {
  id: string;                    // Auto-generated or manual
  playerId: string;

}

9. App-Level Orchestration API (App.tsx)
These are the key callback wiring patterns in App.tsx that connect components:

9.1 Request Flow Callbacks
Top Up (DepositQRScreen onSubmitDeposit):

Input → { amount, paymentMethod, transactionId, receiptImage }
Side effects:
  1. Creates UnitDepositRequest (status: 'pending')
  2. Creates AppNotification (status: 'pending')
  3. Creates Transaction (status: 'pending')
  4. Shows success toast
  5. setTimeout(3000ms):
     - Updates request → status: 'approved'
     - Updates notification → 'approved' title/message
     - Updates transaction → status: 'approved'
     - Calls updateBalance(+amount)
     - Shows approved toast
Buy Units (UnitRequestModal onSubmit):

Input → { amount, paymentMethod, transactionId, paymentScreenshot, note? }
Side effects: Same lifecycle as Top Up
Cash Out (CashoutModal onSubmit):

Input → { amount, paymentMethod, accountInfo, note? }
Side effects:
  1. Creates UnitDepositRequest (type: 'cashout', status: 'pending')
  2. Creates AppNotification (status: 'pending')
  3. Creates Transaction (type: 'cashout', amount: -amount, status: 'pending')
  4. setTimeout(3000ms):
     - All updates same pattern but with cashout messaging
     - Calls updateBalance(-amount)
9.2 Navigation Interception
handleNavigate(screen: string): void
If screen === 'place-bet': opens SessionSelectionModal instead of navigating
Otherwise: calls setCurrentScreen(screen) directly
