Complete Project Blueprint Prompt
## Background

I'm building a Myanmar 2D/3D lottery system (Agent Backend/Dashboard) using Myanmar language (Pyidaungsu font) and MMK currency. This is a React + Tailwind CSS v4 application. The system manages the full hierarchy of Admin → Master → Agent roles for lottery operations including betting, unit management, deposits, withdrawals, bank account management, and player management.

---

## Tech Stack & Dependencies

- **Framework**: React 18.3.1 (pure client-side SPA, no Next.js, no server-side)
- **Styling**: Tailwind CSS v4 (CSS-based config in `theme.css`, no `tailwind.config.js`)
- **Animation**: Motion (imported as `import { motion, AnimatePresence } from 'motion/react'`)
- **Charts**: Recharts (LineChart, BarChart, AreaChart with responsive containers)
- **Icons**: lucide-react
- **Toasts**: sonner (`import { toast } from "sonner"`, `<Toaster position="top-right" richColors closeButton />`)
- **UI Components**: Full shadcn/ui library in `/src/app/components/ui/` (~47 components: accordion, alert-dialog, badge, button, card, dialog, drawer, dropdown-menu, form, input, label, select, sheet, switch, table, tabs, tooltip, etc.)
- **Drag & Drop**: react-dnd + react-dnd-html5-backend
- **Forms**: react-hook-form@7.55.0
- **Date**: date-fns
- **Carousel**: embla-carousel-react, react-slick
- **Masonry**: react-responsive-masonry
- **Popper**: @popperjs/core, react-popper
- **Resizable**: react-resizable-panels
- **MUI** (available): @mui/material, @emotion/react, @emotion/styled, @mui/icons-material
- **Router** (installed but NOT used for routing - app uses state-based navigation): react-router

Key Feature Flows
1. Buy Units (Agent→Master, Master→Admin)
Agent→Master (Desktop): DashboardView.tsx inline modal shows Master's active bank accounts (filtered by parentId) + amount/paymentMethod/transactionId/note → onRequestUnitDeposit

Agent→Master (Mobile): MobileDashboard.tsx uses BuyUnitsPaymentFlow with masterBankAccounts

Master→Admin (Desktop): MasterDashboard.tsx uses BuyUnitsPaymentFlow with adminBankAccounts

Master→Admin (Mobile): MobileMasterDashboard.tsx uses BuyUnitsPaymentFlow with adminBankAccounts

BuyUnitsPaymentFlow 3-step: Select Method → View Account (QR, copy) → Enter Details → Submit

Screenshot upload REMOVED from all Buy Units forms. paymentScreenshot field kept in types and verification drawers only.

2. Player Deposits (Player→Agent)
Agent views in: Desktop DepositsView / Mobile MobileDeposits
Filter pills: all/pending/approved/rejected
Approve: deducts agent balance, adds player balance + transactions
Reject: marks as rejected
3. Unit Deposit Requests
Created by Buy Units flow → auto-routes to parent
Master reviews Agent requests: Desktop MasterDepositsView / Mobile MobileMasterDeposits
Admin reviews Master requests: Desktop WithdrawalsView / Mobile MobileAdminDashboard
Verification: ReceiptVerificationDrawer (mobile) / PaymentVerificationDesk (desktop)
Approve: balance transfer + transaction records
4. Withdrawals
Request form: paymentMethod, accountNumber, accountName, amount, note → auto-routes to parent
Desktop: WithdrawalsView (incoming + own) / Mobile: MobileWithdrawals
Approve: balance transfer + transactions / Reject: with reason
5. Bank Account Management
Admin: BankSettingsView (desktop) / MobileBankSettings (mobile, bank-settings route)
Agent/Master: MobileBankQRManagement (mobile, bank-qr-management route)
Tab 1: Accounts with Active/Inactive toggles
Tab 2: Incoming requests with filter pills + badge counts
Tab 3: Verification drawer
6. Betting System
Pattern types: 20 different patterns for 2D/3D
Mobile: MobileBetEntry (CleanGboard keyboard)
Desktop: SplitBetEntry via BetEntryView
Player-specific: PlayerBetEntryModal (opened from Players list)
Flow: Enter → Cart → Submit → DashboardBets → Optional SavedRecord
Post-bet: Ledger (aggregated by number), Settlement (report), Betting Records (history)
